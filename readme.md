<h1> Azure Data Factory CI/CD Pipeline </h1>

This project demonstrates a complete Azure Data Factory (ADF) implementation with automated CI/CD using Azure DevOps.
The repository contains ADF pipelines, datasets, linked services, triggers, and YAML-based deployment pipelines.

The goal of the project is to show how ADF development, version control, and automated deployments can be managed in an enterprise environment.

<h2> CI/CD Flow </h2>
The CI/CD process is based on Azure DevOps and follows the standard ADF deployment model:

1. Developers work in the collaboration branch (main).
2. ADF publishes ARM templates to the publish branch.
3. Azure DevOps YAML pipelines:
    - Validate the ARM templates
    - Deploy them to test/QA environments
    - Deploy to production using environment-specific parameters
    - Production deployment requires manual approval before execution

The deployment uses:
- ARM templates generated by ADF
- Parameter files for each environment
- Azure DevOps service connections

Key Features
- Version controlled ADF pipelines and configuration
- Automated deployment using YAML pipelines
- Environment based parameterization
- Separation of collaboration and publish branches
- Clean and reproducible deployment process
